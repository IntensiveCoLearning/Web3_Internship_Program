# 2025-08-19

Uniswap V2 学习笔记
1. Uniswap V2 背景
Uniswap 是一个基于以太坊的自动化做市商（AMM, Automated Market Maker）去中心化交易所 (DEX)

V2 相对 V1 的升级：
引入了任意 ERC-20/ERC-20 交易对（V1 只能 ERC-20/ETH）。
支持闪电交换（Flash Swaps）。
定价机制优化，更透明和安全。

2. 核心原理
恒定乘积做市公式
Uniswap V2 的核心公式是：𝑥⋅𝑦=𝑘
x = Token0 的数量，y = Token1 的数量，k = 常数（池子的流动性不变）

当有人用 Token0 换 Token1 时，池子需要调整数量，保持 
𝑘不变，从而形成价格。

交易定价
交易价格由 两边池子余额决定：

𝑃rice=𝑦÷𝑥
​举例：池子里有 100 DAI 和 1 ETH，价格就是 100 DAI/ETH。


手续费机制
每笔交易收取 0.3% 手续费。
手续费会自动添加到流动性池中，作为流动性提供者 (LP) 的奖励。

3. 主要合约逻辑
Factory 合约
用来创建新的交易对 (Pair)。每个交易对对应一个独立的合约。

Pair 合约
管理某个 TokenA/TokenB 的池子。
负责：
存款/取款流动性
Swap 逻辑
定价与储备管理

 Router 合约
用户交互的入口

负责处理：
添加/移除流动性
TokenA <-> TokenB 交换
跨多池的路径选择

4. 流动性提供 (LP) 机制
提供流动性：用户按比例存入 TokenA 和 TokenB，获得 LP Token。
收益：持有 LP Token 的用户，可以分享交易手续费。

无常损失 (Impermanent Loss)：
当两种 Token 价格波动时，LP 的实际收益可能低于单独持有 Token。

5. 特殊功能：闪电交换 (Flash Swaps)
Uniswap V2 的创新点：允许用户 先拿走 Token，再支付或归还，只要交易最后满足资金平衡即可。
应用：套利、借贷、清算。
风险：需要严格保证最后偿还，否则交易回滚。

6. 优缺点总结
优点
完全去中心化、无需许可
支持任意 ERC-20 交易对
手续费收入回馈 LP
支持闪电交换

缺点
无常损失风险
资本利用率较低（需要大量流动性支撑）
价格发现机制依赖套利者

7. 学习重点
读懂 恒定乘积公式 和价格变化机制。
理解 Factory/Pair/Router 合约 的职责分工。
掌握 流动性提供 和 无常损失 的计算方法。
了解 闪电交换 在套利与借贷中的应用。

总结
Uniswap V2 是去中心化交易所的 里程碑版本，其核心思想“恒定乘积 + 无许可流动性池”奠定了 AMM 模式的基础。掌握 V2 的逻辑，有助于理解后续 V3 的集中流动性和更复杂的 DEX 设计。

# 2025-08-18

Web3 链上社交（On-chain Social）

1. 定义与核心理念
链上社交指的是 基于区块链身份和数据 的新型社交模式。区别于 Web2（微信、推特）掌握平台数据，Web3 链上社交的核心是：
身份去中心化：用户以钱包地址/ENS/DID 作为身份。
数据所有权：社交关系、互动数据、内容均存储在链上，用户可自由迁移。
可组合性：任何应用可以基于相同的社交图谱构建新服务。

2. 代表项目
-Lens Protocol（Polygon）
提供去中心化社交图谱，用户可以创建 Profile NFT，帖子、关注、评论都存储在链上。
开发者可直接调用其 Graph API，快速构建 DApp。
生态繁荣，有多个基于 Lens 的应用（Lenster、Phaver 等）。

-CyberConnect（跨链）
强调 去中心化社交图谱 + Gasless 体验。
提供一套 SDK，开发者可以快速接入用户关系。
主打用户“社交资产”可迁移。

-Friend.tech（Base 链）
创新玩法：将社交关系 “金融化”，通过 Key/Share 交易绑定 KOL 与粉丝。
热点性极强，2023 年 TVL 一度突破 2 亿美金。
争议：强金融化导致 “投机属性” 大于社交属性。

-Farcaster
去中心化社交网络，采用 可移植身份，链上注册，但内容在去中心化存储层。
被称为 Web3 版 Twitter。

3. 商业模式
社交资产化
社交关系被代币化（如 Friend.tech 的 Shares）。
粉丝关系转化为直接价值（买卖 Key、NFT 门票、打赏）。
社交数据的二次利用
用户数据开放，任何 DApp 都能基于社交图谱开展二次应用（广告推荐、DAO 招募、空投发放）。
应用内金融化功能
NFT 门票、内容付费、代币激励。
典型模式是 “边社交边赚” (Social-to-Earn)。

4. 运营策划与热点打法
借势热点
如 ETH 升级、Memecoin 爆火时，社交 DApp 可推出“链上身份 + Meme NFT”。
内容激励
给活跃创作者发放代币或 NFT 徽章，形成社区粘性。
KOL 策略
借鉴 Friend.tech，绑定意见领袖，让其带动粉丝迁移。
跨链与工具集成
与钱包、NFT 市场、DeFi 协议合作，形成流量入口。

5. 未来趋势
与 AI 融合：AI Agent 在链上社交中自动生成内容和交互。
与 DeFi 融合：社交资产可直接抵押、借贷、流动性交易。
多链互通：社交关系数据将打破链间壁垒（Cosmos/Polkadot + Ethereum）。
DAO 与治理：链上社交社区可能逐步演化为去中心化自治组织。

总结
链上社交的核心是 把社交关系和数据放在用户手里，并通过 代币化和金融化 带来新的商业模式。
Lens → 社交图谱基础设施
CyberConnect → 多链可组合
Friend.tech → 金融化玩法
Farcaster → 类 Twitter 平替
从运营角度，关键是 抓热点 + KOL 策略 + 长期内容激励。

DApp名称：SkillLink - 去中心化技能证明与学习互助平台
核心目标：
构建一个去中心化的学习与技能认证平台，让用户可以：

技术架构
模块	技术组件	功能说明
前端	React / Next.js + Ethers.js/Web3.js	用户交互界面，钱包连接，任务浏览与提交
合约层	Solidity + Hardhat/Foundry	管理任务创建、技能NFT铸造、奖励结算、声誉记录等逻辑

任务参与：
其他用户接取任务并提交成果（文本、代码、音频等存储于 IPFS）。

社区验证：
通过治理机制投票或引入专家（质押 SLT）进行评分验证。

# 2025-08-09

NFT

定义
-同质化代币（FT，全称 Fungible Token）是可以被任意交换的代币，就像钱一样。

-非同质化代币（NFT，全称 Non-fungible Token）则是独一无二、完全不同，难以用来平等交换的代币。

NFT的特性和价值
-独一无二，不可分割，不可等价交换
-基于区块链技术不可篡改，公开透明
-万物皆可NFT,NFT可在虚拟世界代表任意非钱币的实体
-具体功能/玩法由参与者定义，具有无限想象空间
-永久&完全控制权（大部分数字藏品不算真正的NFT,因为不在公链可被平台控制）
-确权成本低
-无限想象空间


体而言，NFT 市场处于非常早期的阶段，作品质量参差不齐鱼龙混杂。所以在参与购买 NFT 的时候，需要 DYOR（Do your own research)

区块链打工人——项目中的刑事风险  

仅凭招聘信息很难完全完全判断风险
常见罪名：集资诈骗、非法吸收公众存款、组织/领导传销活动，开设赌场，非法经营

一道红线：ICO(Initial Coin Offering)
-中国大陆绝对禁止ICO，也禁止国内募资
-海外发币需持牌、很昂贵，未持牌项目也绝对违法

风险项目：资金汇集（把社会面各个面的资金汇集起来）
-交易所与虚拟货币基金
-是否涉及法币
-海外的合规性是否完备(AML反洗钱与KYC对客户认知)
KYC（Know Your Customer）
· 目的：确认客户身份、评估风险等级、防止匿名或虚假账户。
· 关键动作：
– 身份核验：护照、驾照、人脸识别、活体检测等。
– 地址验证：近期水电账单、银行对账单等。
– 资金来源与用途：了解客户资金从哪来、用来干什么。
– 持续监控：账户行为发生异常时重新评估风险。
AML（Anti-Money Laundering）
· 目的：发现并阻止洗钱、恐怖融资等金融犯罪。
· 关键动作：
– 客户风险评级（CDD/EDD）。
– 交易监控：对大额、频繁、异常交易实时预警。
– 制裁名单筛查：与OFAC、EU、UN 等制裁名单实时比对。
– 可疑交易报告（SAR）：一旦触发阈值，必须向监管机构报告。
– 员工培训与内部审计：确保制度长期有效。

-技术上是否排除中国大陆用户（IP封锁、中国大陆手机不允许注册&登录）

“误入”传销与赌场
-项目扩张过程的合规:回避多级返佣与团队奖励
-非典型案例:合约交易
-NFT的合规玩法设计

难以判断:洗钱与外汇管制
洗钱链条的一环or外汇兑换的媒介·难点:资金的来龙去脉对于非管理层来说难以知晓出现不清楚的情况（例如资金显著大于平均的现金流是，先暂时冻结资金进行调查），提高自身警惕

判断的经验之谈
-项目是否在明确、受监管的司法辖区内注册
-是否为了取得合规资质，经过专业机构的第三方审计或安全测试

Web3合规性要求与常见法律风险

Web3的合规性需适配其去中心化、跨地域等特性，核心要求包括：

	•	监管框架适配：需遵守不同国家/地区对虚拟资产（如加密货币）的分类监管（如是否视为商品、证券、货币），以及反洗钱（AML）、反恐怖融资（CFT）规则，如KYC（了解你的客户）流程、交易记录留存等。

	•	数据合规：用户隐私数据（如钱包地址关联信息）的收集、存储需符合《通用数据保护条例》（GDPR）等数据保护法规，避免未经授权使用或泄露。

	•	智能合约合规：合约代码的合法性（如是否涉及欺诈、非法金融活动）、条款透明度，以及可能引发的民事责任（如代码漏洞导致的用户损失）。

常见法律风险包括：

	•	因跨境运营违反当地金融监管（如未经许可开展代币发行融资）；

	•	虚拟资产交易平台因KYC/AML漏洞被追责；

	•	智能合约漏洞引发的用户维权或监管处罚；

	•	数据泄露导致的合规处罚或民事诉讼。

Web3入职法律风险防范

入职Web3企业时，需从个人权责和法律风险角度做好防范：

	•	明确业务合法性：入职前核实企业核心业务（如代币发行、交易、NFT平台等）是否符合当地监管，避免参与涉嫌非法金融活动的项目，防止因“单位犯罪”承担连带责任。

	•	厘清合同条款：劳动合同中需明确岗位职责（如是否涉及代币发行、合规审查等敏感工作）、薪酬形式（如以加密货币支付时的税务申报责任）、保密义务及违约责任，避免模糊条款导致后续纠纷。

	•	合规培训与记录：主动参与企业合规培训，了解业务涉及的监管红线（如禁止向特定地区用户提供服务），留存培训记录以证明已尽到注意义务。

	•	个人行为边界：避免利用职务便利进行内幕交易、操纵市场等行为，不参与企业超出合规范围的决策，必要时留存书面异议记录。

Web3常见网络安全风险与防护方式

Web3因依赖区块链、智能合约、数字钱包等技术，面临独特的网络安全风险，防护需针对性设计：

常见风险：

	•	私钥泄露：用户钱包私钥因钓鱼攻击、恶意软件或平台漏洞被窃取，导致资产被盗。

	•	智能合约漏洞：如重入攻击、逻辑漏洞等，被黑客利用转移资产（如DAO攻击事件）。

	•	钓鱼与诈骗：伪造项目网站、NFT合约或空投链接，诱骗用户授权转账或泄露信息。

	•	节点攻击：区块链网络的节点被恶意控制（如51%算力攻击），导致交易篡改或双花问题。

	•	供应链攻击：第三方工具（如钱包插件、DApp开发框架）被植入恶意代码，间接危害用户资产。

防护方式：

	•	用户层面：采用硬件钱包存储私钥，避免在公共网络操作；验证DApp/合约地址真实性，拒绝点击不明链接；开启二次验证（2FA）。

	•	企业层面：智能合约上线前进行多轮安全审计（如形式化验证、渗透测试）；部署紧急暂停机制（当检测到异常时冻结合约）；对交易进行实时监控，及时拦截异常转账。

	•	网络层面：区块链节点采用分布式部署，提升抗攻击能力；定期更新节点软件，修补已知漏洞；针对跨链交互场景，强化跨链桥的安全验证逻辑。

	•	生态协作：建立漏洞赏金计划，鼓励白帽黑客发现并上报漏洞；行业共享威胁情报，共同抵御新型攻击手段。

1.区块链概念与结构：区块链是去中心化、不可篡改的分布式账本，典型结构包括区块、链结构、共识机制和加密算法。

2.链类型：

公链：完全开放、人人可参与（如以太坊）

私链：组织内部使用，权限受控

联盟链：多方共同维护，适用于企业协作

3.Web3 内涵：强调用户数据所有权、去中心化身份与资产，依托区块链实现可信交互。

Web3 vs Web 3.0 vs Web2：Web2以平台为中心，Web 3.0关注智能与语义，Web3关注去中心化与用户主权。

4.去中心化优劣：

优势：抗审查、用户自治、安全透明

挑战：扩展性差、治理复杂、用户门槛高

5.稳定币类型：分为法币抵押（如USDT、USDC）、加密资产抵押（如DAI）、算法稳定币（如FRAX），对应不同的稳定机制和风险水平。

6.借贷协议收益模型：平台通过贷款利息、清算罚金、质押代币等获取收益，用户赚取利息或激励。收益受供需关系和利用率影响。

7.Uniswap V3 vs V4：V3引入集中流动性和多池设计，提升资本效率；V4新增“hooks”功能和单合约架构，增强可编程性、降低Gas费。
